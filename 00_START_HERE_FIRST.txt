HOW TO DEPLOY (5 SIMPLE STEPS):

STEP 1: Open Windsurf
────────────────────
Open this folder (Nabda-whatsapp) in Windsurf.

  Terminal: windsurf ./Nabda-whatsapp
  Or: File → Open → Select Nabda-whatsapp


STEP 2: Copy the Windsurf Prompt
────────────────────────────────
1. Find file: WINDSURF_PROMPT.txt (in this folder)
2. Copy ALL the text
3. Paste into Windsurf chat
4. Press Enter


STEP 3: Let Windsurf Verify Everything
───────────────────────────────────────
Windsurf will automatically:
  ✅ Check all files exist
  ✅ Build the project (npm run build)
  ✅ Lint code (check for errors)
  ✅ Commit changes to git
  ✅ Tell you "READY TO DEPLOY"

Wait 2-3 minutes for this to complete.


STEP 4: Push to GitHub
──────────────────────
When Windsurf says "READY TO DEPLOY":

  git push origin main

That's it! Vercel auto-deploys.


STEP 5: Configure Environment
──────────────────────────────
After Vercel deploys (2-3 min):
  1. Go to Vercel Dashboard
  2. Settings → Environment Variables
  3. Add these 8 variables (from your .env file):
     - NABDA_API_TOKEN
     - NABDA_API_URL
     - NABDA_INSTANCE_ID
     - SUPABASE_URL
     - SUPABASE_SERVICE_ROLE_KEY
     - VITE_SUPABASE_URL
     - VITE_SUPABASE_ANON_KEY
     - GEMINI_API_KEY (optional)

WHAT WINDSURF WILL DO:

When you paste WINDSURF_PROMPT.txt, Windsurf will:

1. Verify Files Exist
   ✅ server.ts has personalizeMessage()
   ✅ src/lib/gemini-helper.ts exists (AI)
   ✅ src/components/TemplateABTest.tsx exists (A/B testing)
   ✅ CampaignView.tsx has test mode
   ✅ TemplateEditor.tsx has documentation

2. Build Project
   ✅ npm install (installs dependencies)
   ✅ npm run build (creates dist/ folder)

3. Check Code Quality
   ✅ npm run lint (or tsc --noEmit)
   ✅ No TypeScript errors
   ✅ All imports resolve

4. Commit to Git
   ✅ git add . (stages all files)
   ✅ git commit (creates commit)
   ✅ Shows git status

5. Final Status
   ✅ Confirms "READY TO DEPLOY"
